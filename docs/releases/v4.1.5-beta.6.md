# Release Notes - v4.1.5-beta.6

## What's Changed

### üêõ Critical Bug Fix: Sensor Availability

This release fixes a critical bug where sensors displayed **stale data** at night instead of becoming **unavailable** when the inverter powered down.

#### The Problem

When the inverter shut down at night (or the VSN card became unreachable):
- ‚ùå Sensors kept showing the last values from sunset
- ‚ùå Users saw incorrect data (e.g., "Current Power: 1500W" at midnight)
- ‚ùå No indication that the inverter was offline
- ‚ùå Data appeared current but was actually frozen from hours ago

#### Root Cause

Three locations in `api.py` were returning `False` instead of raising exceptions when connection failed:

1. **`connect()` method (line 511)**: When `check_port()` failed (inverter unreachable)
2. **`async_get_data()` method (line 779)**: When data read failed
3. **`async_get_data()` method (line 783)**: When connection failed

Without exceptions, the coordinator never raised `UpdateFailed`, so Home Assistant:
- Didn't mark entities as unavailable
- Kept displaying stale `coordinator.data` values
- Never indicated the integration was offline

#### The Solution

Changed all three failure points to **raise exceptions** instead of returning `False`:

```python
# Before (Silent Failure):
if await self.check_port():
    # connect...
    return True
return False  # ‚ùå Sensors show stale data

# After (Proper Exception):
if await self.check_port():
    # connect...
    return True
raise VSNConnectionError(...)  # ‚úÖ Sensors become unavailable
```

#### Changes Made

#### Expected Behavior After Fix

**Night (Inverter Offline):**
1. VSN card unreachable ‚Üí `check_port()` times out ‚úÖ
2. `connect()` raises `VSNConnectionError` ‚úÖ
3. Coordinator catches exception ‚Üí raises `UpdateFailed` ‚úÖ
4. Home Assistant marks entities as **unavailable** ‚úÖ
5. Sensors display: **"Unavailable"** instead of stale values ‚úÖ

**Morning (Inverter Online):**
6. VSN card responds ‚Üí connection succeeds ‚úÖ
7. Data read succeeds ‚úÖ
8. Sensors become **available** with fresh data ‚úÖ
9. Coordinator resumes normal updates ‚úÖ

### Technical Details

**Why This Bug Occurred:**

The Home Assistant DataUpdateCoordinator has specific behavior:
- If `async_update_data()` **raises `UpdateFailed`** ‚Üí Entities become **unavailable** ‚úÖ
- If `async_update_data()` **returns any value** (even `False`) ‚Üí `coordinator.data` stays **unchanged** ‚Üí Sensors keep stale values ‚ùå

Our code was returning `False` on connection failures, which triggered the second behavior.

**Flow Comparison:**

```
Old Flow (Buggy):
check_port() ‚Üí timeout
connect() ‚Üí return False
async_get_data() ‚Üí return False
coordinator.async_update_data() ‚Üí return False (no exception!)
coordinator.data ‚Üí unchanged (keeps old values)
sensors ‚Üí display stale data ‚ùå

New Flow (Fixed):
check_port() ‚Üí timeout
connect() ‚Üí raise VSNConnectionError
async_get_data() ‚Üí propagate exception
coordinator.async_update_data() ‚Üí raise UpdateFailed
coordinator ‚Üí marks entities unavailable
sensors ‚Üí display "Unavailable" ‚úÖ
```
